# 智慧社区安防系统 App Web 排版方案（V1）

## 1. 设计目标
- 一屏呈现“监控主能力 + 告警态势 + 运维操作”，减少页面跳转。
- 优先支持值班人员快速判断风险，其次支持运营人员做数据复盘。
- 所有核心模块保持固定入口：Agent 聊天、开放词汇加词、告警统计、天气质量、摄像头控制、个人信息。

## 2. 页面信息架构
页面采用 **三栏仪表台 + 顶部态势栏 + 底部快捷区**：

1. 顶部态势栏（全宽）
   - 系统标题与当前社区名称
   - 在线设备数、处理中告警数、今日告警数、联动成功率
   - 当前时间与运行状态

2. 左栏（智能交互）
   - Agent 聊天窗口（会话列表 + 输入区）
   - 开放词汇加词模块（输入词汇、风险等级、场景标签、提交记录）

3. 中栏（监控核心）
   - 实时监控视频主画面（主摄像头）
   - 多路缩略画面（2~4 个）
   - 摄像头控制面板（云台方向、变焦、预置位、录像/抓拍）

4. 右栏（态势与人员）
   - 告警信息列表（最新事件、级别、位置、处理状态）
   - 告警统计（按类型/时间的简化可视图）
   - 天气与空气质量（温度、湿度、AQI、风速）
   - 用户个人信息（角色、班次、联系方式、快速操作）

5. 底部快捷区（全宽）
   - 一键操作：布防、静音、广播、联动预案
   - 最近 24h 趋势概览

## 3. 视觉与交互方向
- 色彩：深青蓝 + 安全橙红作为告警强调色，避免常见紫色科技风。
- 字体：标题使用 YouSheBiaoTiHei，数值与英文标签使用 Novecento wide。
- 背景：多层径向渐变 + 细网格纹理，增强“指挥中枢”氛围。
- 动效：卡片进入淡入上移、告警条轻微呼吸闪烁、状态灯脉冲。
- 响应式：
  - 桌面端：三栏并行（30% / 40% / 30%）。
  - 平板端：两栏（中栏优先，右栏下移）。
  - 手机端：单栏纵向流，监控主画面固定在前两屏。

## 4. 模块优先级建议
1. 实时监控 + 摄像头控制（最高）
2. 告警信息 + 统计
3. Agent 聊天 + 加词
4. 天气空气 + 个人信息

## 5. 本次产出文件
- `first/index.html`：页面结构
- `first/styles.css`：视觉样式与响应式规则
- `first/main.js`：基础交互（告警状态切换、加词记录、时间刷新）
