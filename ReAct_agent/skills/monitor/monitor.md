# 监控类技能 (Monitor Skills)

## 概述
监控类技能用于查询监控点列表与单点详情。
该技能组通常与告警技能联动，用于定位事件发生位置与责任区域。

## 适用场景
- 查询系统当前可访问的全部监控点。
- 根据监控点名称快速定位点位信息。
- 获取点位状态、负责人、告警次数、视频流信息。
- 在天气、告警等查询前进行点位确认。

## 技能列表

### 1. get_monitor_list - 监控点列表查询
- **功能**: 查询当前用户可访问的监控点列表。
- **参数**:
  - `none`: 无参数。
- **返回说明**:
  - 返回监控点总数与前若干条列表。
  - 每条记录至少包含 `name`、`id`、`department`、`leader`、`running`、`alarmCnt`。

### 2. get_monitor_detail - 监控点详情查询
- **功能**: 按监控点 ID 或名称查询监控点详情。
- **参数**:
  - `monitor_id`: 可选，监控点 ID。
  - `monitor_name`: 可选，监控点名称。
- **返回说明**:
  - 返回单点详情文本，包括基础属性与能力清单。
  - 若未命中，返回“未找到对应的监控点”。

## 参数规范
- `monitor_id` 与 `monitor_name` 至少提供一个。
- 若二者都提供，优先按 `monitor_id` 精确匹配。
- `monitor_name` 支持模糊匹配（名称、区域、编号加权匹配）。

## 匹配策略
- 精确 ID 匹配优先级最高。
- 名称完全匹配优先于包含匹配。
- 候选评分示例：
  - 点位名称匹配权重最高。
  - 区域字段次之。
  - 编号字段再次之。

## 返回文本风格
- 先输出结论，再展示结构化字段。
- 列表场景默认分页展示，防止信息过载。
- 详情场景优先展示：点位名、状态、负责人、告警次数。

## 失败与降级
- 后端不可达时，返回统一失败信息。
- 输入名称过短且歧义大时，建议用户补充关键字。
- 若结果为空，避免编造，直接提示未找到。

## 记忆与上下文
- 查询详情后，系统会记录最近 monitor 的 `id` 与 `name`。
- 后续天气查询可复用该 monitor 上下文，无需用户重复输入。

## 后端接口
- `GET /monitor` - 查询监控点列表。
- `GET /monitor/{id}` - 查询监控点详情（当前实现主要基于列表本地匹配）。

## 调用示例
- `列出所有监控点` -> `get_monitor_list`。
- `查看南门监控点详情` -> `get_monitor_detail`。
- `2号监控点负责人是谁` -> `get_monitor_detail`。

## 扩展约定
- 新增监控类工具时，使用标准标题：`### N. tool_name - 描述`。
- 参数必须写为 `` `param`: 说明 `` 便于自动解析。
- 若涉及上下文复用，需补充“记忆字段”说明。
