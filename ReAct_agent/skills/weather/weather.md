# 天气类技能 (Weather Skills)

## 概述
天气类技能用于查询监控点关联的天气信息。
该技能主要服务于事件研判：例如雨雪、大风、湿度等因素对告警结果的影响。

## 适用场景
- 查询某监控点当前天气，辅助实时值守判断。
- 查询历史天气，辅助复盘某时段告警背景。
- 在未明确点位时，结合会话上下文自动补齐监控点。

## 技能列表

### 1. get_weather_newest - 最新天气查询
- **功能**: 查询指定监控点的最新天气。
- **参数**:
  - `monitor_id`: 可选，监控点 ID。
  - `monitor_name`: 可选，监控点名称。
- **返回说明**:
  - 返回温度、湿度、天气现象、采集时间。

### 2. get_weather_history - 历史天气查询
- **功能**: 查询指定监控点的历史天气记录。
- **参数**:
  - `monitor_id`: 可选，监控点 ID。
  - `monitor_name`: 可选，监控点名称。
  - `time_text`: 可选，自然语言时间，如 `昨天`、`近7天`。
- **返回说明**:
  - 返回历史记录列表，可按时间范围过滤。

## 参数规范
- `monitor_id` 与 `monitor_name` 至少一个。
- 无监控点参数时，可从最近上下文补齐。
- `time_text` 支持自然语言，不要求固定格式。

## 时间语义
- `昨天` -> 昨日全天。
- `近3天` -> 包含当天在内最近 3 天。
- `近7天` -> 包含当天在内最近 7 天。
- `近30天` -> 包含当天在内最近 30 天。

## 匹配与解析
- 点位解析复用监控技能的匹配逻辑。
- 时间解析复用系统统一时间窗口解析逻辑。
- 当 monitor 未命中时，返回明确提示而非猜测。

## 返回文本风格
- 最新天气：一句话结论 + 核心指标。
- 历史天气：列表化展示，默认展示前 10 条。
- 结果为空时说明“暂无天气数据”。

## 失败与降级
- 后端不可达时返回统一失败信息。
- 时间窗口无效时回退到默认范围，不中断流程。
- 点位未匹配时提示用户补充监控点名称或 ID。

## 记忆与上下文
- 查询天气后，系统会记录最近 monitor 的 `id` 与 `name`。
- 后续用户可直接说“这个点位昨天的天气”，系统会自动补齐。

## 后端接口
- `GET /weather/newest/{monitor_id}` - 查询最新天气。
- `GET /weather/all/{monitor_id}` - 查询历史天气。

## 调用示例
- `1号监控点现在天气怎么样` -> `get_weather_newest`。
- `南门监控点近7天天气` -> `get_weather_history`。
- `这个点位昨天什么天气` -> 上下文补齐后 `get_weather_history`。
