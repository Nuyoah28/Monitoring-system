# 告警类技能 (Alarm Skills)

## 概述
告警类技能用于查询、统计、追踪和更新告警数据。
该技能组是系统中最核心的数据能力，支持列表、计数、详情、趋势与状态更新。

## 适用场景
- 值班人员查看当前未处理告警。
- 运营人员统计某类事件在一段时间内的数量。
- 处置人员根据告警 ID 查看详细信息并回写处理状态。
- 管理人员查看近 7 天或近 30 天趋势，评估风险变化。

## 技能列表

### 1. get_alarm_list - 告警列表查询
- **功能**: 查询告警列表，支持类型、状态、等级和时间筛选。
- **参数**:
  - `case_types`: 可选，int 数组，例如 `[5]` 表示明火。
  - `status`: 可选，`0=未处理`，`1=已处理`。
  - `warning_levels`: 可选，int 数组，例如 `[4, 5]`。
  - `page_size`: 可选，返回条数，默认 `10`。
  - `time_text`: 可选，自然语言时间，如 `今天`、`近7天`。
- **返回说明**:
  - 返回标准化文本列表，包含告警 ID、事件名、监控点、区域、等级、状态、时间。
  - 当结果超过 `page_size` 时，显示前 N 条并提示剩余数量。

### 2. get_alarm_count - 告警数量统计
- **功能**: 统计符合筛选条件的告警数量。
- **参数**:
  - `case_types`: 可选，int 数组，例如 `[5]`。
  - `status`: 可选，`0=未处理`，`1=已处理`。
  - `warning_levels`: 可选，int 数组，例如 `[4, 5]`。
  - `time_text`: 可选，自然语言时间，如 `今天`、`近7天`。
- **返回说明**:
  - 返回统计值与筛选条件摘要。

### 3. get_alarm_detail - 告警详情查询
- **功能**: 按告警 ID 查询告警详情。
- **参数**:
  - `alarm_id`: 必填，告警 ID。
- **返回说明**:
  - 返回单条告警的完整信息；若不存在则返回明确提示。

### 4. get_alarm_history - 历史告警查询
- **功能**: 查询告警趋势和历史变化。
- **参数**:
  - `defer`: 可选，`1/3/7/30`，分别表示今天、近 3 天、近 7 天、近 30 天。
- **返回说明**:
  - 返回趋势摘要，包括总量趋势、区域分布、类型分布。

### 5. get_realtime_alarm - 实时告警
- **功能**: 查询实时告警概况和大屏态势数据。
- **参数**:
  - `none`: 无参数。
- **返回说明**:
  - 返回总告警、今日新增、日变化与类型分布。

### 6. update_alarm_status - 更新告警状态
- **功能**: 更新告警处理状态，可标记为已处理或未处理。
- **参数**:
  - `alarm_id`: 必填，告警 ID。
  - `status`: 必填，`0=未处理`，`1=已处理`。
  - `processing_content`: 可选，处理说明。
- **返回说明**:
  - 返回更新结果确认文本。

## 参数规范
- `case_types` 和 `warning_levels` 推荐传数组，即使只有 1 个值。
- `time_text` 优先使用自然语言，由系统统一解析时间窗口。
- `status` 只接受 `0` 或 `1`，其他值会被拒绝。
- `page_size` 建议范围 `1~50`。

## 时间语义
- `今天` -> 当日 00:00:00 到 23:59:59。
- `昨天` -> 昨日全天。
- `近3天` -> 包含当天在内最近 3 天。
- `近7天` -> 包含当天在内最近 7 天。
- `近30天` -> 包含当天在内最近 30 天。

## 返回文本风格
- 优先给结论，再给细节。
- 所有数字带单位（条）。
- 列表默认按时间倒序。

## 失败与降级
- 后端不可达时，返回统一失败提示，不抛出堆栈。
- 参数缺失时返回可执行修复建议。
- 单条详情不存在时返回 `未找到 ID` 明确提示。

## 记忆与上下文
- 系统会记录最近一次告警工具调用参数。
- 系统会记录最近出现的告警 ID 列表。
- 当用户说“上一个告警”“刚才那个告警”时，可自动补齐 ID。

## 后端接口
- `GET /alarm/query` - 分页查询告警列表。
- `GET /alarm/query/cnt/history` - 查询历史告警统计。
- `GET /alarm/realtime` - 查询实时告警。
- `GET /alarm/{alarm_id}` - 查询告警详情。
- `PUT /alarm/update` - 更新告警状态。

## 调用示例
- `今天有多少未处理明火告警` -> `get_alarm_count`。
- `近7天四级和五级告警列表` -> `get_alarm_list`。
- `查看告警 12 详情` -> `get_alarm_detail`。
- `把告警 12 标记为已处理，备注：已通知物业` -> `update_alarm_status`。
- `近30天告警趋势` -> `get_alarm_history`。

## 扩展约定
- 新增告警工具时，必须沿用 `### N. tool_name - 描述` 格式。
- 参数项必须使用 `` `param` `` 语法，便于自动解析。
- 若存在上下文依赖，需在文档中写明“记忆字段”。
